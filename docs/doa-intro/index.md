# DOA 入门

本章只做一件事：建立你对阵列接收、来波方向和空间扫描响应的第一层直觉。

## 学习目标

- 理解什么是波达方向估计
- 知道均匀线阵为什么是课程的起点
- 认识波长、阵元间距和角度之间的关系
- 运行第一份空间响应实验并解释结果

## 先修要求

- 会运行基本的 Python 命令
- 知道复数的模是什么意思
- 对向量和矩阵有最基本印象

如果这些还不熟，也不影响你先跑实验。先把图跑出来，再补概念。

## 核心概念

### 1. 什么是 DOA

DOA 是 `Direction of Arrival`，也就是“信号从哪个方向来”。  
如果你只有一个传感器，通常只能测到“有信号”；但如果你有一排传感器，就能利用不同传感器接收到的相位差来推断信号方向。

### 2. 为什么从均匀线阵开始

均匀线阵 `ULA` 的好处是结构简单：

- 每个阵元沿一条直线排开
- 相邻阵元间距固定
- 很多经典 DOA 方法都可以从这里起步

### 3. 为什么经常使用半波长间距

如果阵元间距太大，扫描图里容易出现额外峰值，也就是常说的栅瓣。  
入门阶段先用 `lambda/2`，能减少很多不必要的困扰。

## 运行步骤

```powershell
python examples/doa_intro_array_scan.py
```

也可以显式改参数：

```powershell
python examples/doa_intro_array_scan.py --angle 30 --num-sensors 12 --spacing-scale 0.5
```

## 实验观察

建议按顺序观察这三件事：

### 观察 1：改变真实角度

把 `--angle` 从 `20` 改到 `-25`。  
你应该看到主峰位置整体跟着移动。

### 观察 2：增加阵元数

把 `--num-sensors` 从 `8` 改到 `12` 或 `16`。  
你通常会看到主峰更尖，这意味着角度区分能力在增强。

### 观察 3：增大阵元间距

把 `--spacing-scale` 从 `0.5` 改到 `0.75` 甚至 `1.0`。  
你需要观察是否出现额外峰值，这就是后面学习传统方法时必须重视的阵列设计问题。

## 练习

1. 为什么单个天线通常无法直接估计来波方向？
2. 当 `--num-sensors` 增大时，扫描曲线为什么会变尖？
3. 为什么入门阶段推荐先使用 `lambda/2` 的阵元间距？

## 参考答案

1. 因为单个天线缺少空间采样维度，无法形成不同阵元之间的相位差比较。
2. 因为阵列孔径变大后，对不同角度的响应区分更明显，主瓣通常更窄。
3. 因为半波长间距更容易避免栅瓣，让初学者先建立稳定的空间响应直觉。
